1. System Modules

1) Ingestión

Parsers/OCR, normalización, metadatos, selección de jurisdicción.

2) Razonamiento

NER + mapeo semántico → requisitos aplicables y jerarquía normativa.

3) Adaptación

Reglas transformadoras, plantillas parametrizadas, resolución de conflictos.

4) Trazabilidad

Bitácora de reglas, referencias, hashes de evidencia y motivos legibles.

5) Evidencia

Reportes imprimibles, anexos estructurados, export JSON/PDF.

6) Export

Persistencia, descarga y vínculo a commit/artefactos.

4. Interfaces (prototype stubs)

CLI (pseudo)

$ iclvs ingest --file contrato.pdf --jur EU,ES
$ iclvs reason --in dataset.json --rules ruleset.json
$ iclvs adapt  --in requirements.json --tpl templates/
$ iclvs report --run 2025-10-27 --out report.pdf

HTTP (pseudo)

POST /v0/review
{ "text": "...", "jurisdictions":["EU","ES"] }

200 OK
{ "decision":"ADJUST","applicableRules":[...], "trace":{...} }
